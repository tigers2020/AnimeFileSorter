# 애니메이션 파일 정리 애플리케이션 - 1년차 2분기 개발 계획

## 개발 목표
1년차 2분기는 기본 프로젝트 구조를 확립하고 초기 프로토타입 개발에 집중합니다. 특히 확장 가능한 플러그인 아키텍처의 기반을 마련하여 다양한 메타데이터 소스 및 외부 서비스를 유연하게 활용할 수 있도록 합니다. **또한 데이터베이스 중심의 데이터 관리 전략을 구체화하여 모든 설정, 메타데이터, 플러그인 데이터를 일관된 방식으로 데이터베이스에 저장하는 구조를 구현합니다.**

> **중요**: 모든 개발 단계에서 변경사항 적용 후 프로그램을 실행하여 기능이 정상적으로 작동하는지 확인해야 합니다. 어떤 변경이든 전체 시스템 안정성을 훼손해서는 안 됩니다.

## 산출물 명세

### 코어 모듈 및 프로젝트 구조
- `src/anime_file_sorter/`: 기본 프로젝트 구조 (초기 패키지 및 모듈 구성)
  - `__init__.py`: 패키지 초기화 및 버전 정보 (v0.2.0)
  - `config.py`: 전역 설정 및 상수 정의
  - `core/`: 핵심 기능 모듈
    - `__init__.py`: 모듈 초기화
    - `hash.py`: ED2K 해시 계산 인터페이스
    - `hash_basic.py`: 기본 해시 계산 구현
    - `hash_mt.py`: 멀티스레딩 해시 계산 구현
    - `file_utils.py`: 파일 처리 유틸리티
    - `logging.py`: 로깅 시스템
  - `plugins/`: 플러그인 시스템 기반 구조
    - `__init__.py`: 모듈 초기화
    - `manager.py`: 플러그인 관리자 구현
    - `interface.py`: 플러그인 인터페이스 정의
    - `loader.py`: 플러그인 검색 및 로딩 구현
    - `registry.py`: 플러그인 레지스트리 구현
  - `data/`: 데이터 관리 계층
    - `__init__.py`: 모듈 초기화
    - `db.py`: 데이터베이스 관리 모듈
    - `session.py`: 세션 관리 및 트랜잭션 처리
    - `migration/`: 마이그레이션 스크립트
      - `__init__.py`: 마이그레이션 시스템 초기화
      - `manager.py`: 마이그레이션 관리자
      - `versions/`: 마이그레이션 버전 스크립트
    - `cache.py`: 데이터 캐싱 시스템
  - `ui/`: UI 컴포넌트
    - `__init__.py`: 모듈 초기화
    - `main_window.py`: 메인 애플리케이션 윈도우
    - `app.py`: 애플리케이션 진입점
    - `components/`: UI 컴포넌트 모음
      - `__init__.py`: 컴포넌트 모듈 초기화
      - `file_browser.py`: 파일 브라우저 컴포넌트
      - `metadata_view.py`: 메타데이터 표시 컴포넌트
      - `progress_bar.py`: 진행 상태 표시 컴포넌트
      - `settings_panel.py`: 설정 패널 컴포넌트
    - `styles/`: UI 스타일 관련 모듈
      - `__init__.py`: 스타일 모듈 초기화
      - `theme.py`: 테마 관리 시스템
      - `stylesheets/`: CSS 스타일시트 파일
        - `light.css`: 라이트 테마 스타일시트
        - `dark.css`: 다크 테마 스타일시트
